## 2. 字段定义规范

### 2.1 字段类型选择

| 数据类型 | 使用场景 | 示例 |
|----------|----------|------|
| BIGINT | 主键、金额 | id, tax_amount |
| VARCHAR(n) | 字符串、固定长度编码 | code, name |
| DECIMAL(p,s) | 精确金额 | DECIMAL(18,2) |
| DATETIME | 时间戳 | create_time |
| DATE | 日期 | rzsgrq |
| TEXT | 长文本 | error_msg |
| MEDIUMTEXT | 中等长文本 | request_param |

### 2.2 金额字段精度

| 字段类型 | 精度 | 说明 |
|----------|------|------|
| 普通金额 | DECIMAL(18,2) | 适用于大部分金额字段 |
| 税率字段 | DECIMAL(10,4) | 保留4位小数 |
| 比例字段 | DECIMAL(5,2) | 百分比，保留2位小数 |

| 约束 | 说明 |

### 3.2 外键约束

## 5. 索引设计原则

### 5.1 索引选择

| 索引类型 | 使用场景 |
|----------|----------|
| UNIQUE | 唯一性约束字段、业务主键 |
| INDEX | 查询条件字段、排序字段 |
| 联合索引 | 多字段组合查询 |

### 5.2 索引设计原则

- WHERE条件字段建立索引
- 避免在区分度低的字段建索引（如：性别、状态）
- 组合索引遵循最左前缀原则
- 避免冗余索引
