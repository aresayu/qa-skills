# E2E 测试用例模板

> 适用范围：基于产品文档/用户故事设计端到端（End-to-End）场景，用真实用户旅程验证跨模块协作  
> 输出目标：覆盖关键业务闭环（主流程+异常分支），作为发布前回归核心集；可标注自动化候选（Playwright/Cypress 等）

---

## 1. 用例元信息

- **用例编号**：E2E-<业务域>-<三位序号>（例：E2E-CHECKOUT-001）
- **用例标题**：用户在【前置条件】下完成【业务目标】，预期【业务结果/系统状态】
- **业务域/模块**：注册登录 / 下单支付 / 退款售后 / 账户权限 / 内容发布 / …
- **需求/用户故事/缺陷关联**：PRD-xxx / US-xxx / BUG-xxx
- **用例类型**：主流程 / 异常流程 / 边界流程 / 回归 / 冒烟
- **优先级**：P0 / P1 / P2
- **风险等级**：High / Medium / Low
- **是否自动化候选**：是 / 否（原因：稳定性/价值/维护成本/依赖外部系统）
- **测试方式**：手工 / 自动化 / 混合
- **作者 / 日期**：
