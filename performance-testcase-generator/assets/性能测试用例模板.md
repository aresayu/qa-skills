# 性能测试用例模板

> 适用范围：基于性能指标/SLA/容量规划，对接口或关键业务链路做基线、稳态、峰值、加压等测试场景设计与执行记录  
> 输出目标：可执行（参数明确）、可复现（数据与环境明确）、可判定（准入准出清晰）、可诊断（监控与证据齐全）

---

## 1. 用例元信息

- **用例编号**：PERF-<场景/模块>-<三位序号>（例：PERF-LOGIN-001）
- **场景名称**：在【负载模型】下压测【接口/链路】，验证【指标目标】
- **需求/指标关联**：SLA-xxx / PRD-xxx / NFR-xxx
- **被测对象**：
  - **接口/链路**：API / E2E 路径 / 批处理任务
  - **环境**：DEV / TEST / STAGING
  - **版本/构建号**：
- **场景类型**：基线 / 稳态负载 / 峰值尖峰 / 逐步加压直到失败 / 链路压测
- **优先级**：P0 / P1 / P2
- **风险等级**：High / Medium / Low
- **作者 / 日期**：

## 2. 环境与前置条件（Preconditions）

- **环境说明**：
  - 部署拓扑（单机/多实例/集群）：
  - 资源规格（CPU/内存/实例数/DB规格）：
  - 配置（连接池/超时/重试/缓存开关）：
- **数据量级**：
  - 用户量/订单量/内容量（写清数量级）：
  - 索引/缓存预热策略：
- **依赖服务**：
  - 外部服务是否 Mock：
  - 降级策略是否开启：

---

## 3. 负载模型（Load Model）

- **并发/用户数**：N
- **目标吞吐**：QPS/TPS（如有）
- **压测时长**：T 分钟（含预热）
- **预热时间**：T 分钟
- **流量模式**：
  - 稳态：固定并发/固定QPS
  - 递增：每 X 分钟 +Y 并发/ +Z QPS
  - 尖峰：在 T 秒内从 A → B 并发
- **请求分布（若多接口/多步骤）**：
  - 接口A：xx%
  - 接口B：xx%
  - 步骤C：xx%

---

## 4. 测试步骤（Steps）

1. 准备环境与数据（按第 2 节）
2. 配置压测脚本/工具（k6/JMeter/Locust/Artillery 等），确认参数与场景
3. 执行预热
4. 执行正式压测（记录开始/结束时间、版本、配置）
5. 采集监控数据与日志（按第 6 节）
6. 输出结论：是否通过、瓶颈点、优化建议

---

## 5. 指标目标与判定（Pass/Fail Criteria）

### 5.1 响应时间（RT）

- **P95**：< ___ ms
- **P99**：< ___ ms
- **Avg**：< ___ ms

### 5.2 吞吐与成功率

- **目标QPS/TPS**：≥ ___
- **成功率**：≥ ___ %
- **错误率**：≤ ___ %（区分 4xx/5xx）

### 5.3 资源与容量（按需）

- **CPU**：≤ ___ %
- **内存**：≤ ___ %
- **DB/缓存**：连接数、慢查询、命中率等阈值

### 5.4 准入/准出

- **准入条件**：环境稳定、依赖可用、数据准备完成
- **通过条件（准出）**：满足以上 RT/QPS/错误率/资源阈值
- **失败条件**：明确写出触发阈值（如 P99>xxx、5xx>y% 等）

---

## 6. 监控与诊断（Observability）

> 目标：出现性能拐点时能定位瓶颈方向。

- **应用指标**：
  - 关键接口 RT 分布（P50/P90/P95/P99）
  - 5xx/4xx 分布与错误码Top
  - 线程池/连接池使用率（如有）
- **资源指标**：
  - CPU / Memory / Disk IO / Network
- **数据库/缓存**（如适用）：
  - 慢查询、锁等待、连接数
  - 缓存命中率、热点key
- **链路追踪**（如有）：
  - traceId 样本、耗时Top span

---

## 7. 测试数据与脚本信息

- **脚本/工具**：k6/JMeter/Locust/...
- **脚本路径/仓库**：
- **参数化策略**：账号池、token刷新、数据唯一性
- **数据清理策略**：是否需要、如何清理

---

## 8. 实际执行记录（Execution）

- **执行人 / 日期 / 环境 / 版本**：
- **实际负载**：并发 N、QPS、时长
- **实际结果**：Pass / Fail / Blocked
- **关键输出**：
  - RT（P95/P99/Avg）：
  - 吞吐（QPS/TPS）：
  - 错误率（4xx/5xx）：
  - 资源峰值（CPU/内存/IO）：
- **证据**：压测报告链接/截图、监控面板链接、日志/trace 样本
- **结论与建议**：瓶颈点、需要的优化、回归建议
