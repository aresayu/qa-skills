# 接口测试用例规范

本文件定义**接口测试**场景下的测试点设计维度。与SKILL配合使用，生成XMind测试用例时按以下规范补充测试点。

---

## 一、必须覆盖的维度

### 1.1 请求与参数

- **正常请求**：必填参数齐全、类型与格式正确、在文档约定范围内的取值
- **参数校验**：
  - 每个必填参数至少1条「缺失」测试点
  - 每个有类型或格式约束的参数（如数字、枚举、日期、手机号、邮箱）：至少1条类型错误或格式非法测试点
  - 有取值范围或长度限制的：用边界值策略设计测试点
- **可选参数**：覆盖「不传」「传空」「传有效值」等

### 1.2 响应与错误码

- **成功响应**：验证HTTP状态码、业务码、响应体结构
- **错误码**：每个错误码至少1条测试点，明确触发条件与预期返回
- **未定义错误**：对明显非法请求，预期应为4xx或约定错误码

### 1.3 鉴权与权限

- **鉴权**：未带鉴权、鉴权无效或过期、鉴权格式错误
- **权限**：越权访问（如A访问B的资源）

### 1.4 幂等与并发

- **幂等**：重复提交（相同幂等键/请求体）测试点
- **并发与限流**：超并发或超频请求测试点

---

## 二、测试点表述格式

在XMind中按以下格式呈现：

```
- 接口名称
  - 正常请求_参数合法
  - 参数校验_必填参数缺失
  - 参数校验_参数类型错误
  - 参数校验_参数格式非法
  - 参数校验_参数边界值
  - 响应_成功返回验证
  - 响应_错误码验证
  - 鉴权_未带token
  - 鉴权_token无效
  - 权限_越权访问
  - 幂等_重复提交
  - 并发_超并发请求
```
