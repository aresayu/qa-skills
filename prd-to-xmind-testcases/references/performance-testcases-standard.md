# 性能测试用例规范

本文件定义**性能测试**场景下的测试点设计维度。与SKILL配合使用，生成XMind测试用例时按以下规范补充测试点。

---

## 一、必须覆盖的维度

### 1.1 指标与基线

- **响应时间**：RT要求（P95、P99、平均）
- **吞吐量**：QPS/TPS或并发用户数
- **资源与容量**：单机/集群承载、连接数、内存

### 1.2 场景类型

- **基准/基线**：低负载下验证接口或关键路径可正常响应
- **稳态负载**：正常/峰值负载下持续运行，验证RT、成功率、错误率
- **峰值/尖峰**：短时流量突增，验证系统不崩溃、降级策略生效
- **压测直到失败**：逐步加压直到系统出现性能拐点或崩溃

### 1.3 关键链路

- **核心接口**：文档中标注的核心接口/关键路径
- **依赖链路**：涉及后端依赖（DB、缓存、外部服务）的链路压测

### 1.4 监控与诊断

- **监控指标**：CPU、内存、IO、网络
- **性能瓶颈**：数据库查询、锁竞争、连接池等

---

## 二、测试点表述格式

在XMind中按以下格式呈现：

```
- 性能测试
  - 基准测试_低负载响应验证
  - 稳态负载_正常负载RT验证
  - 稳态负载_峰值负载验证
  - 稳态负载_错误率验证
  - 峰值测试_流量突增验证
  - 峰值测试_降级策略验证
  - 峰值测试_恢复时间验证
  - 压测直到失败_最大承载量
  - 资源监控_CPU使用率
  - 资源监控_内存使用率
  - 资源监控_IO使用率
  - 瓶颈分析_数据库查询
  - 瓶颈分析_锁竞争
  - 瓶颈分析_连接池
```
