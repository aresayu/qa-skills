# 重新触发算税

**请求方式**：POST  
**请求地址**：`/api/v2/tax/taxCompute/retry`  
**接口描述**：对状态为"处理失败"（status=-9）的算税记录重置状态并重新发起算税。支持重试全部失败记录（不传 `recordIds`）或指定记录重试（传入 `recordIds`）。

> 接口安全（加密 + 签名）规则统一参考 [README · 接口安全](../README.md#4接口安全说明)。

---

## 请求参数

| 字段名称 | 字段含义 | 字段类型 | 必填 | 说明 |
| :--- | :--- | :--- | :---: | :--- |
| cpId | 租户ID | Integer | 是 | - |
| taxClassCode | 报税方案编码 | String | 是 | 关联 wa_tax_class.code |
| sdyf | 所得月份 | String | 是 | 格式：YYYYMM |
| recordIds | 指定重试的记录ID列表 | Array\<Long\> | 否 | 不传则重试该月所有失败记录；传入则仅重试指定记录 |

---

## 响应参数

无业务数据字段，`success=true` 即表示操作成功。

解密后内容：

```json
{
  "message": "操作成功"
}
```

---

## 示例

**请求示例（重试所有失败记录）**

```json
{
  "cpId": 1001,
  "taxClassCode": "TC2025001",
  "sdyf": "202503"
}
```

**请求示例（重试指定记录）**

```json
{
  "cpId": 1001,
  "taxClassCode": "TC2025001",
  "sdyf": "202503",
  "recordIds": [100001, 100002]
}
```

**响应示例（解密后）**

```json
{
  "message": "操作成功"
}
```
