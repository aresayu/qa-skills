# 算税汇总统计

**请求方式**：POST  
**请求地址**：`/api/v2/tax/taxCompute/result/summary`  
**接口描述**：查询指定报税方案、所得月份的算税汇总统计信息，包括总人数、收入总额、应补退税总额及各状态人数分布，常用于展示算税进度概览。

> 接口安全（加密 + 签名）规则统一参考 [README · 接口安全](../README.md#4接口安全说明)。

---

## 请求参数

| 字段名称 | 字段含义 | 字段类型 | 必填 | 说明 |
| :--- | :--- | :--- | :---: | :--- |
| cpId | 租户ID | Integer | 是 | - |
| taxClassCode | 报税方案编码 | String | 是 | 关联 wa_tax_class.code |
| sdyf | 所得月份 | String | 是 | 格式：YYYYMM |

---

## 响应参数

解密后 data 字段：

| 字段名称 | 字段含义 | 字段类型 | 说明 |
| :--- | :--- | :--- | :--- |
| totalPersonCount | 总人数 | Integer | 该月全部算税记录数 |
| totalSre | 收入总额 | BigDecimal | 所有记录本期收入之和 |
| totalYbtse | 应补(退)税总额 | BigDecimal | 正数为应补税总额，负数为应退税总额 |
| successCount | 计算成功人数 | Integer | status=0 的记录数 |
| failedCount | 计算失败人数 | Integer | status=-9 的记录数 |
| computingCount | 计算中人数 | Integer | 正在处理的记录数 |
| notComputedCount | 未计算人数 | Integer | status=-1 的记录数 |

---

## 示例

**请求示例**

```json
{
  "cpId": 1001,
  "taxClassCode": "TC2025001",
  "sdyf": "202503"
}
```

**响应示例（解密后）**

```json
{
  "totalPersonCount": 100,
  "totalSre": 1500000.00,
  "totalYbtse": 45000.00,
  "successCount": 95,
  "failedCount": 2,
  "computingCount": 1,
  "notComputedCount": 2
}
```
