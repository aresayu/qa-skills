# 导出算税结果

**请求方式**：POST  
**请求地址**：`/api/v2/tax/taxCompute/result/export`  
**接口描述**：按条件导出指定报税方案、所得月份的全量算税结果（不分页），支持按算税状态和所得项目代码过滤，常用于将算税结果回写至调用方系统。

> 接口安全（加密 + 签名）规则统一参考 [README · 接口安全](../README.md#4接口安全说明)。

---

## 请求参数

| 字段名称 | 字段含义 | 字段类型 | 必填 | 说明 |
| :--- | :--- | :--- | :---: | :--- |
| cpId | 租户ID | Integer | 是 | - |
| taxClassCode | 报税方案编码 | String | 是 | 关联 wa_tax_class.code |
| sdyf | 所得月份 | String | 是 | 格式：YYYYMM |
| status | 算税状态过滤 | Integer | 否 | `-9`=处理失败 / `-1`=未处理 / `0`=正确 / `1`=有提示信息；不传则导出全部 |
| sdxm | 所得项目代码过滤 | String | 否 | 如 `0101`，不传则导出全部所得类型 |

---

## 响应参数

解密后 data 字段：

| 字段名称 | 字段含义 | 字段类型 | 说明 |
| :--- | :--- | :--- | :--- |
| total | 导出总数 | Integer | - |
| list | 算税结果列表 | Array | 每条记录字段见下方 |

### list 单条记录字段

| 字段名称 | 字段含义 | 字段类型 | 说明 |
| :--- | :--- | :--- | :--- |
| id | 算税记录ID | Long | - |
| xm | 姓名 | String | - |
| zzhm | 证照号码 | String | 后 4 位保留，其余脱敏 |
| sdxm | 所得项目代码 | String | - |
| pch | 批次号 | Integer | - |
| sre | 本期收入 | BigDecimal | - |
| ybtse | 应补(退)税额 | BigDecimal | 正数为应补税，负数为应退税 |
| status | 算税状态 | Integer | `-9`=处理失败 / `-1`=未处理 / `0`=正确 / `1`=有提示信息 |
| isMultiPay | 多次发薪合并 | Integer | `0`=否 / `1`=是 |
| errorinfo | 错误信息 | String | 失败时税局返回的错误提示 |
| computeTime | 计算完成时间 | String | ISO 8601 格式 |

---

## 示例

**请求示例（导出全部）**

```json
{
  "cpId": 1001,
  "taxClassCode": "TC2025001",
  "sdyf": "202503"
}
```

**请求示例（只导出计算成功的工资薪金）**

```json
{
  "cpId": 1001,
  "taxClassCode": "TC2025001",
  "sdyf": "202503",
  "status": 0,
  "sdxm": "0101"
}
```

**响应示例（解密后）**

```json
{
  "total": 95,
  "list": [
    {
      "id": 100001,
      "xm": "张三",
      "zzhm": "310***1234",
      "sdxm": "0101",
      "pch": 1,
      "sre": 15000.00,
      "ybtse": 1500.00,
      "status": 0,
      "isMultiPay": 0,
      "errorinfo": null,
      "computeTime": "2025-03-20T10:30:00"
    }
  ]
}
```
