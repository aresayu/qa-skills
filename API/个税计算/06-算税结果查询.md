# 算税结果查询

**请求方式**：POST  
**请求地址**：`/api/v2/tax/taxCompute/result/list`  
**接口描述**：分页查询指定报税方案、所得月份下的算税结果列表，支持按算税状态过滤。算税完成前可轮询此接口获取最新计算状态。

> 接口安全（加密 + 签名）规则统一参考 [README · 接口安全](../README.md#4接口安全说明)。

---

## 请求参数

| 字段名称 | 字段含义 | 字段类型 | 必填 | 说明 |
| :--- | :--- | :--- | :---: | :--- |
| cpId | 租户ID | Integer | 是 | - |
| taxClassCode | 报税方案编码 | String | 是 | 关联 wa_tax_class.code |
| sdyf | 所得月份 | String | 是 | 格式：YYYYMM |
| status | 算税状态过滤 | Integer | 否 | `-9`=处理失败 / `-1`=未处理 / `0`=正确 / `1`=有提示信息；不传则查全部 |
| pageNum | 页码 | Integer | 否 | 默认 1 |
| pageSize | 每页大小 | Integer | 否 | 默认 20 |

---

## 响应参数

解密后 data 字段：

| 字段名称 | 字段含义 | 字段类型 | 说明 |
| :--- | :--- | :--- | :--- |
| total | 总记录数 | Long | - |
| list | 结果列表 | Array | 每条记录字段见下方 |

### list 单条记录字段

| 字段名称 | 字段含义 | 字段类型 | 说明 |
| :--- | :--- | :--- | :--- |
| id | 算税记录ID | Long | 可用于查询详情 |
| xm | 姓名 | String | - |
| zzhm | 证照号码 | String | 后 4 位保留，其余脱敏 |
| sdxm | 所得项目代码 | String | 税局编码，如 `0101`=正常工资薪金 |
| pch | 批次号 | Integer | 同人同月多次发薪时的批次编号 |
| sre | 本期收入 | BigDecimal | - |
| ybtse | 应补(退)税额 | BigDecimal | 正数为应补税，负数为应退税 |
| status | 算税状态 | Integer | `-9`=处理失败 / `-1`=未处理 / `0`=正确 / `1`=有提示信息 |
| isMultiPay | 多次发薪合并 | Integer | `0`=否 / `1`=是 |
| errorinfo | 错误信息 | String | 失败时税局返回的错误提示 |
| computeTime | 计算完成时间 | String | ISO 8601 格式 |

---

## 示例

**请求示例**

```json
{
  "cpId": 1001,
  "taxClassCode": "TC2025001",
  "sdyf": "202503",
  "pageNum": 1,
  "pageSize": 20
}
```

**响应示例（解密后）**

```json
{
  "total": 100,
  "list": [
    {
      "id": 100001,
      "xm": "张三",
      "zzhm": "310***1234",
      "sdxm": "0101",
      "pch": 1,
      "sre": 15000.00,
      "ybtse": 1500.00,
      "status": 0,
      "isMultiPay": 0,
      "errorinfo": null,
      "computeTime": "2025-03-20T10:30:00"
    }
  ]
}
```
