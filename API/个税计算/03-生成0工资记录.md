# 生成 0 工资记录

**请求方式**：POST  
**请求地址**：`/api/v2/tax/salaryData/zeroWage`  
**接口描述**：为指定人员写入本期收入为 0 的工资薪金（所得项目 `0101`）记录，生成后算税状态为"未处理"，可直接触发算税。

> 接口安全（加密 + 签名）规则统一参考 [README · 接口安全](../README.md#4接口安全说明)。

---

## 请求参数

| 字段名称 | 字段含义 | 字段类型 | 必填 | 说明 |
| :--- | :--- | :--- | :---: | :--- |
| cpId | 租户ID | Integer | 是 | - |
| taxClassCode | 报税方案编码 | String | 是 | 关联 wa_tax_class.code |
| sdyf | 所得月份 | String | 是 | 格式：YYYYMM |
| personIds | 人员 ID 列表 | Array\<String\> | 是 | 证照号码列表，不能为空 |

---

## 响应参数

无业务数据字段，`success=true` 即表示操作成功。

外层结构：

```json
{
  "success": true,
  "data": {
    "body": "AES加密的响应（16进制）"
  }
}
```

解密后内容：

```json
{
  "message": "操作成功"
}
```

---

## 示例

**请求示例**

```json
{
  "cpId": 1001,
  "taxClassCode": "TC2025001",
  "sdyf": "202503",
  "personIds": [
    "310101199001011234",
    "310101199002022345"
  ]
}
```

**响应示例（解密后）**

```json
{
  "message": "操作成功"
}
```
