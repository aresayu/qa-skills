# 触发算税

**请求方式**：POST  
**请求地址**：`/api/v2/tax/taxCompute/trigger`  
**接口描述**：触发指定报税方案、所得月份下的算税任务。支持批量（全员）和单人两种触发模式，以及 SKJS（上月已申报）和 GYSS（上月未申报，隔月试算）两种算税模式。系统会自动检测同月多批次 `0101` 工资薪金并合并后统一提交。

> 接口安全（加密 + 签名）规则统一参考 [README · 接口安全](../README.md#4接口安全说明)。

---

## 请求参数

| 字段名称 | 字段含义 | 字段类型 | 必填 | 说明 |
| :--- | :--- | :--- | :---: | :--- |
| cpId | 租户ID | Integer | 是 | - |
| taxClassCode | 报税方案编码 | String | 是 | 关联 wa_tax_class.code |
| sdyf | 所得月份 | String | 是 | 格式：YYYYMM |
| triggerType | 触发类型 | String | 是 | `BATCH`=批量算税（全员）；`PERSON`=单人算税 |
| personIds | 单人算税人员列表 | Array\<String\> | 条件必填 | `triggerType=PERSON` 时必填，传证照号码列表 |
| computeMode | 算税模式 | String | 是 | `SKJS`=个税计算（上月已申报）；`GYSS`=隔月试算（上月未申报），默认 `GYSS` |

> **说明**：
> - `SKJS` 模式：上月已完成申报时使用，系统从税局获取上月累计数据；
> - `GYSS` 模式：上月未申报时使用，系统自动从数据库查询上月所得数据作为累计基础；
> - 只有状态为"未处理"（status=-1）的记录会被触发，已计算成功的记录不会重复触发。

---

## 响应参数

外层统一封装，解密后 data 字段：

| 字段名称 | 字段含义 | 字段类型 | 说明 |
| :--- | :--- | :--- | :--- |
| taskId | 算税任务ID | String | 可用于后续轮询，可能为空（无可算税记录时） |
| totalCount | 本次触发人数 | Integer | 实际提交算税的记录数 |
| mergedCount | 多次发薪合并人数 | Integer | 检测到同月多批次 0101 并合并的人员数量 |

---

## 示例

**请求示例（批量算税）**

```json
{
  "cpId": 1001,
  "taxClassCode": "TC2025001",
  "sdyf": "202503",
  "triggerType": "BATCH",
  "computeMode": "SKJS"
}
```

**请求示例（单人算税）**

```json
{
  "cpId": 1001,
  "taxClassCode": "TC2025001",
  "sdyf": "202503",
  "triggerType": "PERSON",
  "personIds": ["310101199001011234"],
  "computeMode": "GYSS"
}
```

**响应示例（解密后）**

```json
{
  "taskId": "task-20250301-abc123",
  "totalCount": 50,
  "mergedCount": 2
}
```
