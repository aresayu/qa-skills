# 人员报送管理

**请求方式**：POST
**请求地址**：`/api/v2/tax/person/{action}`
**接口描述**：人员报送管理是指对个人所得税申报人员进行管理，包括人员信息的新增、修改、删除、报送、并档等操作。每个扣缴义务人下有多个申报人员。

> 接口安全（加密 + 签名）规则统一参考 [README · 接口安全](../README.md#4接口安全说明)。

## 场景说明

### 1. 新增人员报送
企业有新增人员，需要将人员进行信息采集并报送登记。税局会接受报送信息并验证人员的真实性，并返回报送状态和验证状态。报送状态是税局是否成功接收报送信息的状态，验证状态是验证人员是否真实的状态。

### 2. 离职后重新报送
若员工原来是报送成功的，然后填写离职日期，离职后，需要重新报送人员，告知税局该人员的离职时间。

### 3. 信息变更重新报送
若变更员工的其他信息，如任职受雇类型、任职受雇时间、手机号码、联系地址等，都需要重新报送人员，告知税局人员信息变更后的情况。
