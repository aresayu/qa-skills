- Base URL: `/api/tax-declaration/external`
- Content-Type: `application/json`
- 公共响应结构: 外层为通用 `Result`，业务数据位于 `data.data`

## 公共响应结构

```json
{
  "success": true,
  "code": 200,
  "message": "success",
  "data": {
    "data": {}
  }
}
```

说明:

- `success`: 控制器层是否成功返回
- `code` / `message`: 通用返回包装字段，具体值取决于公共 `Result` 实现
- `data.data`: 当前接口的业务响应体

## 1. 申报数据查询

- Method: `POST`
- Path: `/query`

### 请求参数

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `taxClassCode` | string | 否 | 报税方案编码（`wa_tax_class.code`） |
| `cpId` | integer | 是 | 租户 ID |
| `sdyf` | string | 是 | 所得月份，格式 `YYYY-MM` |
| `categoryCode` | string | 否 | 所得大类编码 |
| `pageNum` | integer | 否 | 页码，默认 `1` |
| `pageSize` | integer | 否 | 每页大小，默认 `20`，最大 `200` |

### 请求示例

```json
{
  "taxClassCode": "TAX_CLASS_001",
  "cpId": 1001,
  "sdyf": "2026-03",
  "categoryCode": "01",
  "pageNum": 1,
  "pageSize": 20
}
```

### 业务响应体 `data.data`

类型: `PageResult<DeclarationQueryResponse>`

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `list` | array | 列表数据 |
| `total` | long | 总记录数 |
| `pageNum` | integer | 当前页码 |
| `pageSize` | integer | 每页大小 |

`list` 元素结构:

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `taxClassCode` | string | 报税方案编码 |
| `nsrsbhName` | string | 扣缴义务人名称 |
| `nsrsbh` | string | 纳税人识别号 |
| `sdyf` | string | 所得月份 |
| `categoryCode` | string | 所得大类编码 |
| `taxPayers` | integer | 纳税人数 |
| `currentIncome` | number | 本期收入 |
| `totalIncome` | number | 累计收入 |
| `taxAmount` | number | 应纳税额 |
| `taxSupplement` | number | 应补(退)税额 |
| `declarationType` | string | 申报类型 |
| `declarationStatus` | string | 申报状态 |

### 响应示例

```json
{
  "success": true,
  "data": {
    "data": {
      "list": [
        {
          "taxClassCode": "TAX_CLASS_001",
          "nsrsbhName": "某某公司",
          "nsrsbh": "91310000XXXXXXXX",
          "sdyf": "2026-03",
          "categoryCode": "01",
          "taxPayers": 28,
          "currentIncome": 120000.00,
          "totalIncome": 350000.00,
          "taxAmount": 5600.00,

      "pageSize": 20
    }
  }

```

## 2. 个税申报提交

- Method: `POST`
- Path: `/submit`

### 请求参数

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `taxClassCode` | string | 是 | 报税方案编码（`wa_tax_class.code`） |
| `cpId` | integer | 是 | 租户 ID |
| `sdyf` | string | 是 | 所得月份，格式 `YYYY-MM` |
| `categoryCode` | string | 是 | 所得大类编码 |

### 请求示例

```json
{
  "taxClassCode": "TAX_CLASS_001",
  "cpId": 1001,
  "sdyf": "2026-03",
  "categoryCode": "01"
}
```

### 业务响应体 `data.data`

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `status` | string | `SUCCESS` / `PROCESSING` / `FAILED` |
| `message` | string | 响应说明 |
| `submissionId` | string | 申报 ID，实际为 `wa_tax_declaration.declaration_biz_id` |

### 响应示例

```json
{
  "success": true,
  "data": {
    "data": {
      "status": "SUCCESS",
      "message": "个税计算已受理，申报任务将在计算成功后自动发起",
      "submissionId": "DECL202603010001"
    }
  }
}
```

### 说明

- 该接口不是同步完成申报。
- 当前实现是先触发“申报前个税计算”，算税成功后再由异步回调继续发起正式申报。
- `submissionId` 供 `/status` 接口查询使用。

## 3. 申报详情查询

- Method: `POST`
- Path: `/detail`

### 请求参数

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `taxClassCode` | string | 是 | 报税方案编码 |
| `cpId` | integer | 是 | 租户 ID |
| `sdyf` | string | 是 | 所得月份，支持 `YYYY-MM` 或 `YYYYMM` |
| `categoryCode` | string | 是 | 所得大类编码（如 `01` / `02` / `03`） |
| `pageNum` | integer | 否 | 页码，默认 `1` |
| `pageSize` | integer | 否 | 每页大小，默认 `20`，最大 `200` |

### 请求示例

```json
{
  "taxClassCode": "TAX_CLASS_001",
  "cpId": 1001,
  "sdyf": "2026-03",
  "categoryCode": "01",
  "pageNum": 1,
```

## 4. 申报状态查询

- Method: `POST`
- Path: `/status`

### 请求参数

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `submissionId` | string | 是 | 申报 ID，即 `wa_tax_declaration.declaration_biz_id` |

### 请求示例

```json
{
  "submissionId": "DECL202603010001"
}
```

### 业务响应体 `data.data`

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `status` | string | `SUCCESS` / `PROCESSING` / `FAILED` |
| `message` | string | 响应说明 |
| `submissionId` | string | 原申报 ID |

### 状态映射

| `wa_tax_declaration.declaration_status` | 接口返回 `status` | 说明 |
| --- | --- | --- |
| `0` | `PROCESSING` | 未申报 |
| `1` | `PROCESSING` | 申报处理中 |
| `2` | `SUCCESS` | 申报成功 |
| `3` | `FAILED` | 申报失败 |

### 响应示例

```json
{
  "success": true,
  "data": {
    "data": {
      "status": "PROCESSING",
      "message": "个税申报处理中",
      "submissionId": "DECL202603010001"
    }
  }
}
```

## 备注

- `sdyf` 在服务内部会被标准化为 `YYYYMM` 后再查询数据库。
- `/status` 当前是 `POST` + JSON body，不是路径参数形式。
- 申报状态枚举:
  - `0`: 未申报
  - `1`: 申报处理中
  - `2`: 申报成功
  - `3`: 申报失败
  - `4`: 作废
